Melee Combat System

Overview

A focused melee combat prototype demonstrating animation-driven attacks, combat state management, directional blocking, dodge mechanics with animation-synchronized invulnerability frames, guard break interactions, and a modular ability system with resource management in a multiplayer environment.

Demo

Gameplay Demonstration

Core Systems

  • 4-hit light attack combos with animation-synchronized hit windows
  • Stamina, health, and mana systems with action-based drain/regeneration
  • Directional dodging and blocking with i-frames and stamina tracking
  • Guard break mechanics and enemy hit/death reactions
  • Modular ability system integrated with combat states, cooldowns, and resource costs
  • Server-authoritative hit detection, ability validation, and dodge/block logic
  • Enemy AI using finite state machines for behaviors and reactions

Animation & Integration

Combat actions, dodge frames, and guard break reactions are synchronized with animations to ensure accurate hit detection and smooth state transitions. The architecture is transferable to 3D environments and engines like Unity or C++ projects.

Event & Progression System

A modular system tracks player/enemy states, quest flags, and world interactions. It enables seamless integration of new content without altering core combat logic.

Challenges & Solutions

  • Preventing exploits: Server-side validation ensures attacks, abilities, dodges, and blocks follow combat rules.
  • Responsive abilities: Input buffering, state machine constraints, and resource gating maintain responsiveness while preventing unintended cancels.

Future Improvements

  • Expanded enemy AI and behavior complexity
  • Enhanced animation blending and keyframe transitions
  • Additional resource-based abilities and combo-heavy attack variations

Takeaways

  • I learned how to design a combat system that feels responsive without sacrificing server authority or fairness.
  • I gained experience integrating animation events with gameplay logic to synchronize attacks, dodges, and guard breaks.
  • I understood the importance of modular systems for abilities, AI, and event handling for future scalability.
  • Balancing resources like stamina and mana requires careful tuning and playtesting to make gameplay engaging and strategic.
  • Debugging multiplayer interactions taught me the value of testing and thinking about edge cases for exploits and animation desyncs.
