Melee Combat System Prototype (Roblox)

Overview

A focused melee combat prototype built to demonstrate animation-driven attacks, combat state management, stamina usage, and basic enemy AI in a multiplayer environment.

Demo

Gameplay video coming soon

Core Features

  • 4-hit light attack combo system
  • Animation-based hit windows
  • Stamina drain and regeneration
  • Enemy hit reactions and death

Systems Breakdown

  • Combat state machine (Idle, Attacking, Recovery)
  • Combo input buffering and reset timing
  • Server-authoritative hit detection
  • Enemy AI using finite state machine logic

Technical Details

  • Roblox Lua (client/server separation)
  • RemoteEvents for combat replication
  • Modular script architecture
  • Server validation to prevent exploit damage

Challenges & Solutions

Problem: Preventing players from applying damage through spoofed client events.
Solution: Server validates attack state, timing window, and hit results before applying damage.

What I’d Improve Next

  • Expanded enemy behaviors
  • Improved animation blending
