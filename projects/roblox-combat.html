Melee Combat System Prototype (Roblox)

Overview

A focused melee combat prototype built to demonstrate animation-driven attacks, combat state management, directional blocking, dodge mechanics with animation-synchronized invulnerability frames, guard break interactions, and a modular ability system with mana/resource management in a multiplayer environment.

Core Features

  • 4-hit light attack combo system with animation-synchronized hit windows
  • Stamina, health, and mana systems with action-based drain and regeneration
  • Directional dodging with i-frames validated on the server
  • Enemy hit reactions, guard break states, and death sequences

Systems Breakdown

  • Combat state machine (Idle, Attacking, Recovery, Ability Use)
  • Combo input buffering and reset timing for responsive attack chains
  • Server-authoritative hit detection and ability validation to prevent exploits
  • Enemy AI using finite state machine logic for behavior and reactions

Extended Mechanics

  • Directional blocking system with stamina-based damage mitigation and guard stamina tracking
  • Heavy attacks that apply increased stamina damage to blocking targets and trigger guard break states when block stamina is depleted
  • Guard break transitions enemies into vulnerable reaction states, enabling follow-up attacks
  • Dodge mechanic with animation-synchronized invulnerability frames (i-frames), stamina cost, and recovery timing

Ability System Integration

Implemented a modular ability system designed to integrate directly with the core combat framework, allowing abilities to interact with combat states, stamina, mana costs, cooldowns, and enemy reactions.

  • Abilities define activation conditions, resource cost, cooldown, and combat state transitions
  • Activation is validated against combat state, stamina/mana availability, and cooldown timers
  • Abilities can modify damage behavior, movement, or trigger enemy reactions
  • Mana regeneration and ability lockout prevent ability spam and invalid overlaps

Event and Progression System

Implemented a modular event management system to track player and enemy states, flags for quests or scripted sequences, and interactions between player abilities and world events. This system allows seamless integration of new content without modifying core combat logic.

Animation Integration

Combat actions, dodge frames, and guard break reactions are synchronized with animation events to ensure accurate hit detection and state transitions. The system is designed to be transferable to 3D environments with keyframe-based animations.

Technical Details

  • Roblox Lua (client/server separation, modular script architecture)
  • RemoteEvents for secure combat and ability replication
  • State machines for combat, abilities, and enemy AI
  • Server validation for hit detection, ability execution, and dodge/block logic to prevent exploits
  • Designed with transferable architecture suitable for C++ or Unity 3D implementations

Challenges & Solutions

Problem: Preventing players from applying damage or bypassing combat rules through spoofed client events.
Solution: Implemented server-side validation for attacks, abilities, dodge, and block/guard break states synchronized with animation events.

Problem: Ensuring abilities and dodge mechanics feel responsive without allowing unintended animation cancel exploits.
Solution: Input buffering, state machine constraints, and resource gating (stamina/mana) maintain responsiveness while enforcing valid transitions.

What I’d Improve Next
