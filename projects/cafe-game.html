Yokai Cafe

Overview

Yokai Cafe is a first-person horror café game where players juggle normal work tasks while something feels increasingly wrong. The game is structured around a full workday using a phase-based system (opening, serving, closing) that controls pacing, progression, and when unsettling events can occur.

It’s built with a modular, data-driven approach using ScriptableObjects, making it easy to iterate on systems, add new mechanics, and experiment with horror elements layered on top of everyday gameplay.

Core Features

  • Phase-Based Gameplay Loop: The café day is split into opening, serving, and closing phases, which control available interactions and help pace both gameplay and horror.
  • Customer Ordering System: Customers place randomized orders that the player must prepare and deliver correctly to keep the café running.
  • Interaction System: A unified system for interacting with doors, light switches, items, and other world objects.
  • Item Behavior System: Items perform context-specific actions when used, allowing new items to be added without rewriting core logic.
  • ScriptableObject-Driven Design: Data and behavior are separated to keep systems flexible, reusable, and easy to iterate on.

Challenges & Solutions

Keeping systems manageable as scope grew
→ Solved by separating data and logic using ScriptableObjects, reducing hard-coded behavior.

Managing gameplay flow across different phases
→ Built a centralized phase manager to handle transitions and avoid scattered state checks.

Making routine tasks feel tense instead of repetitive
→ Designed core mechanics to support horror events without interrupting normal gameplay flow.

Future Improvements

  • More horror events tied to specific phases
  • Deeper customer behaviors and order variations
  • Persistent consequences for player mistakes
  • Dynamic lighting and audio systems
  • Save/load support and difficulty tuning

Takeaways

  • System-driven design makes iteration faster and safer
  • Separating data from logic helps projects scale cleanly
  • Horror works best when layered onto familiar mechanics
  • Strong foundations early make future features easier to add
