Melee Combat System Prototype (Roblox)

Overview

A focused melee combat prototype built to demonstrate animation-driven attacks, combat state management, blocking with stamina-based damage mitigation, directional dodging with animation-synchronized invulnerability frames, and integrated ability usage in a multiplayer environment.

Demo

Gameplay video coming soon

Core Features

  • 4-hit light attack combo system
  • Animation-based hit windows
  • Stamina, health, and mana systems with action-based drain and regeneration
  • Enemy hit reactions and death

Systems Breakdown

  • Combat state machine (Idle, Attacking, Recovery)
  • Combo input buffering and reset timing
  • Server-authoritative hit detection
  • Enemy AI using finite state machine logic

Extended Mechanics

  • Directional blocking system with stamina-based damage mitigation and guard stamina tracking
  • Heavy attacks that deal increased block stamina damage and trigger guard break states when block stamina is depleted
  • Guard break transitions enemies into vulnerable reaction states, enabling follow-up attacks

Ability System Integration

Implemented a modular ability system designed to integrate directly with the core combat framework, allowing abilities to interact with combat states, stamina, mana costs, cooldowns, and enemy reactions.

  • Abilities define activation conditions, mana cost, cooldown, and combat state transitions
  • Ability activation is validated against current combat state and available resources
  • Abilities can modify damage behavior, apply movement overrides, or trigger enemy reaction states
  • Mana regeneration and ability lockout prevent ability spam and invalid overlaps

Technical Details

  • Roblox Lua (client/server separation)
  • RemoteEvents for combat replication
  • Modular script architecture
  • Server validation to prevent exploit damage

Challenges & Solutions

Problem: Preventing players from applying damage through spoofed client events.
Solution: Server validates attack state, timing window, and hit results before applying damage.

What I’d Improve Next

  • Expanded enemy behaviors
  • Improved animation blending
