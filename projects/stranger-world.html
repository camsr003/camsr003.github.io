Stranger World Game

Game Link

Overview

"Stranger World" is an open-world roleplay game playable on Roblox, a game based on the Netflix show "Stranger Things". This project involved contributing to a game with a somewhat established codebase. I was responsible for integrating new gameplay mechanics alongside UI and backend systems, which required rewiring and refactoring existing logic to support new interactions, data flow, and player states.

Contributions

  • Designed and implemented new UI logic that integrated with existing backend systems
  • Rewired and extended legacy code to support new mechanic-driven behaviors and state changes
  • Integrated and adapted free-to-use, open-source modules to fit the game’s architecture and design constraints
  • Refactored portions of existing systems to improve compatibility, readability, and maintainability
  • Ensured new mechanics worked seamlessly with pre-existing progression, data storage, and event systems

Challenges & Solutions

  • Existing systems were not originally designed with these new mechanics in mind, making direct implementation risky without breaking existing functionality: I analyzed how current systems handled player state, events, and data flow, then extended those systems rather than replacing them. This allowed new mechanics to plug into established logic while preserving stability.
  • Third-party modules did not always align with the game’s existing architecture: I isolated external modules and modified their interfaces where necessary, allowing them to integrate cleanly without leaking implementation details into core systems.
  • Enemy attack coordination: Implemented an "Attack Queue" module that tells the NPC when it is their turn to attack, creating less combat pressure for the player.

Future Improvements

  • Further modularize gameplay mechanics to improve reuse and scalability
  • Adding debug tools and visualizers
  • Making more user-friendly mechanics to boost accessibility

Takeaways

  • Learned how to safely introduce new gameplay mechanics into an existing production codebase
  • Improved ability to reason about and modify legacy systems without destabilizing them
  • Gained experience coordinating gameplay, backend, and UI logic across multiple systems
  • Developed stronger judgment around when to refactor versus extend existing code
  • Reinforced the importance of architectural consistency in collaborative game development
